Hands-On Work

01 / FYP · COMPUTER FORENSICS
ForensicEye — AI-Driven Digital Forensics Platform

Final Year Project (Air University). An AI-powered desktop forensics platform that combines traditional forensic tools with LLMs and RAG to enable natural language evidence querying, semantic artifact search, attack chain reconstruction, and automated court-ready report generation.

Python 3.11 Llama 3.1 LlamaIndex (RAG) Autopsy + Sleuth Kit Volatility3 YARA-X Neo4j Qdrant PyQt6 SQLite

// Key Steps

  • Built evidence ingestion module with SHA-256/MD5 chain-of-custody logging
  • Integrated Autopsy, bulk_extractor & Volatility3 for artifact extraction
  • Implemented RAG pipeline (LlamaIndex + Qdrant) to ground LLM answers in real evidence
  • Built Neo4j attack-chain graph for cross-artifact relationship mapping
  • Added YARA-X malware scanning and AI threat explanation module
  • Designed PyQt6 desktop GUI with AI chat, artifact browser & timeline tabs
  • Generated court-ready PDF reports via ReportLab with full audit trail
Learned: How to architect a full forensic pipeline from evidence ingestion to AI-assisted analysis — and why RAG is essential to prevent hallucination in legal-grade AI tools. Compliant with ISO/IEC 27037, NIST SP 800-86, and ACPO Guidelines.
02 / PEN TESTING
Web Application Penetration Test

Conducted a structured pentest against a deliberately vulnerable web application (DVWA) to practice identifying and exploiting common OWASP Top 10 vulnerabilities.

Burp Suite Nmap DVWA SQLMap

// Key Steps

  • Enumerated open ports and service versions with Nmap
  • Intercepted and modified HTTP requests via Burp Suite
  • Exploited SQL Injection and XSS vulnerabilities
  • Attempted CSRF and broken authentication attacks
  • Compiled a pentest report with CVSS severity ratings
Learned: How attackers chain web vulnerabilities, and how to write actionable remediation recommendations for developers.
03 / MALWARE
Basic Malware Analysis (Static & Dynamic)

Analyzed a malware sample in an isolated VM environment using both static and dynamic techniques to understand its behavior, persistence mechanisms, and C2 communication.

PEStudio Wireshark Process Monitor REMnux

// Key Steps

  • Performed static analysis: PE headers, strings, imports
  • Set up an isolated FlareVM sandbox environment
  • Monitored file system and registry changes at runtime
  • Captured C2 traffic patterns using Wireshark
  • Mapped behavior to MITRE ATT&CK framework TTPs
Learned: The difference between static and dynamic analysis, and how malware hides itself through common evasion techniques.
